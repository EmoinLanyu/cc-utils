image_scan Trait¶

Attributes

name required? default type explanation
parallel_jobs no 12 int amount of parallel scanning threads
cve_threshold no 7 int CVE threshold to interpret as an error
filters no {‘include_image_references’: (), ‘exclude_image_references’: ()} FilterCfg optional filters to restrict container images to process
processing_mode no force_upload str Protecode processing mode
reference_protecode_group_ids no () str an optional list of protecode group IDs to import triages from.
protecode_group_id yes None int technical protecode group id to upload to
protecode_cfg_name no None str protecode cfg name to use (see cc-utils)
email_recipients no () str optional email recipients to be notified about critical scan results
upload_registry_prefix no None str if specified, all matching container images are also uploaded as copies to the specified container registry. The original image reference names are mangled.

filters Attributes

name required? default type explanation
include_image_references no () str a list of regular expressions. If configured, only matching image references are processed. By default, all image references are considered.
exclude_image_references no () str a list of regular expressions. If configured, matching image references are exempted from processing. Has precedence over include_image_references. By default, no image references are excluded.
